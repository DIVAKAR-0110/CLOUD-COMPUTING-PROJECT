🧠 Performance Analysis Report

What is CPU Spike?

A CPU spike occurs when processor utilization suddenly jumps above 80% for a short period. This is detected in real-time and logged automatically.

Common Causes of CPU Spikes

  • 🔁 Runaway processes — Infinite loops, memory leaks
  • 📦 Batch jobs — Scheduled tasks running simultaneously
  • 🌐 Traffic bursts — Sudden increase in HTTP requests
  • 🗄️ Database queries — Heavy unoptimized queries
  • 🔒 Cryptographic operations — SSL handshakes, hashing

How CloudMonitor Detects Spikes

1 Poll system metrics every 2 seconds via systeminformation
2 Stream data to dashboard via Socket.IO WebSocket
3 If CPU > 80%, flag as isSpiking: true
4 Persist spike record to MongoDB Atlas
5 Fire visual alert with timestamp & top process info
